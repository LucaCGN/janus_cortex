## Purpose
Reference docs explain what the system is for, what is already true, and which constraints should remain stable across planning waves.

Use reference docs for:
- product intent
- stable research hypotheses
- current validated system state
- durable schema, API, and analysis contracts

- branch sequencing
- active execution packets
- subphase checklists
- dependency ordering and merge strategy
